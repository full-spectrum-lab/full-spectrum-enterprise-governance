# Case: Refund Over-Commitment

## Summary

An AI customer-service system gives a direct refund or compensation-style commitment before authority, evidence, and rule alignment have been properly checked.

## Why this matters

This is one of the clearest early governance scenarios because a single sentence can create:

- user expectation
- perceived platform promise
- merchant cost pressure
- follow-up trust loss if the promise is later withdrawn

## Governance question

Was the AI merely being polite, or did it create an unauthorized enterprise commitment?

## Signals to inspect

- explicit promise wording
- authority source
- order state
- applicable platform rule
- merchant rule
- user escalation trajectory

## Output expectation

The governance runtime should help produce:

- RiskAlert
- AuditTrace
- review recommendation
- report for internal QA or supervisor review
